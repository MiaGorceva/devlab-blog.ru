Топ-12 open-source ERP и бизнес-систем, которые реально используют

«Open-source ERP» — это зонтичный термин. Для одних команд это означает установить готовый пакет и настроить модули. Для других — взять платформу и собрать бизнес-логику вокруг собственных процессов.

Здесь — практический обзор: что у каждого варианта получается хорошо, где начинает «кусаться» сложность, и как выбрать подход без трёх месяцев демо и совещаний.

Что на самом деле означает «open-source ERP»

«Open-source ERP» — слишком общий термин. Под ним скрываются как готовые системы, которые можно установить и начать использовать, так и платформы, на которых ERP получается как результат сборки бизнес-логики.

Поэтому здесь не рейтинг «кто лучше», а практичный разбор: что каждая система делает хорошо, где начинаются ограничения и для каких команд она реально подходит.

Три подхода, которые важно не путать

  • Готовые ERP-сюиты — быстрое внедрение, меньше свободы.
  • ERP-платформы — больше гибкости, выше требования к команде.
  • Конструкторы бизнес-систем — low-code как фундамент набора внутренних систем.

Топ-12 open-source ERP и бизнес-систем

1) Odoo (Community Edition)

Odoo — самый узнаваемый open-source ERP. Огромная экосистема модулей, интеграций и партнёров делает его частым стартовым выбором для малого и среднего бизнеса.

Основная проблема проявляется при глубокой кастомизации: система быстро обрастает логикой, которую сложно обновлять и поддерживать.

Подходит для: SMB/SME, модульного внедрения, типовых процессов.
Не лучший выбор, если: процессы часто меняются и требуют глубокой переработки модели.
Links: Site · GitHub

2) ERPNext

ERPNext — цельный open-source ERP с понятной моделью данных. Часто выбирается командами, которым нужен единый пакет без enterprise-сложности.

Лучше всего работает, когда бизнес готов подстроиться под структуру системы, а не наоборот.

Подходит для: компаний с относительно стандартными процессами.
Не лучший выбор, если: бизнес-модель нестабильна и требует постоянных изменений.
Links: Site · GitHub

3) MyCompany (lsFusion)

MyCompany — это platform-first подход к ERP. Вместо жёсткой структуры предлагается набор готовых модулей, которые можно развивать поверх декларативной бизнес-платформы lsFusion.

Такой подход особенно удобен там, где процессы часто меняются и система должна эволюционировать вместе с бизнесом, а не сопротивляться изменениям.

Дополнительное преимущество — естественная совместимость с AI-инструментами: декларативная логика проще для анализа, объяснения и автоматизации.

Подходит для: команд, строящих ERP как живую систему с долгим горизонтом изменений.
Не лучший выбор, если: нужен полностью готовый ERP «из коробки без доработок».
Links: Project site · GitHub · lsFusion platform

4) Dolibarr

Dolibarr — простой ERP/CRM с долгой историей. Часто используется в малом бизнесе благодаря низкому порогу входа.

Его сила — в простоте, его ограничение — в масштабируемости и гибкости.

Подходит для: небольших компаний, базовых ERP+CRM сценариев.
Не лучший выбор, если: требуются сложные цепочки процессов и автоматизация.
Links: Site · GitHub

5) Tryton

Tryton — модульный ERP-фреймворк с аккуратной архитектурой. Часто выбирается техническими командами, ценящими чистоту модели.

Подходит для: инженерных внедрений, модульной сборки ERP.
Не лучший выбор, если: нужен быстрый запуск без команды разработки.
Links: Site · GitHub

6) iDempiere

iDempiere — ERP с выраженным enterprise-наследием. Мощный, но требовательный к внедрению и сопровождению.

Подходит для: сложных операций, зрелых IT-команд.
Не лучший выбор, если: нужна простота и быстрый старт.
Links: Site · GitHub

7) metasfresh

metasfresh ориентирован на операционные и логистические процессы.

Подходит для: складов, supply chain, операций.
Не лучший выбор, если: нужен универсальный ERP без логистического фокуса.
Links: Site · GitHub

8) Axelor

Axelor сочетает ERP и workflow-ориентированный подход.

Подходит для: компаний с большим количеством согласований и процессов.
Не лучший выбор, если: нужен минималистичный ERP без BPM.
Links: Site · GitHub

9) Apache OFBiz

Apache OFBiz — фундамент для построения кастомных бизнес-систем.

Подходит для: компаний с сильной разработкой.
Не лучший выбор, если: нет ресурсов на кастомную разработку.
Links: Site · GitHub

10) NocoBase

NocoBase — low-code платформа для сборки внутренних систем управления.

Подходит для: быстрого создания внутренних бизнес-приложений.
Не лучший выбор, если: нужен готовый ERP-сюит.
Links: Site · GitHub

11) SuiteCRM

SuiteCRM — CRM-first система, которая часто становится ядром операций.

Подходит для: компаний с фокусом на продажи и сервис.
Не лучший выбор, если: ERP-функции — основной приоритет.
Links: Site · GitHub

12) Openbravo

Openbravo часто используется в ритейле и POS-сценариях.

Подходит для: розницы, омниканальных операций.
Не лучший выбор, если: нужен универсальный ERP без retail-фокуса.
Links: Site

Как выбирать без лишней боли

  1. Нужно быстро: берите готовый ERP и ограничьте кастомизацию.
  2. Процессы уникальны: выбирайте платформы.
  3. ERP — это набор систем: low-code честнее монолита.
